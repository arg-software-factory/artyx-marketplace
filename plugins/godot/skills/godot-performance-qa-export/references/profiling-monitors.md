# Profiling and monitors

Capture a repeatable scene, input sequence, build, and target device. Use the Debugger profiler and
Monitors to identify CPU frame work, rendering work, memory, and object growth. Compare a baseline
and a single changed build; deep profiling or editor metrics can distort timing, so confirm in an
exported development build.

## Official sources

- Performance monitors: https://docs.godotengine.org/en/4.6/tutorials/scripting/debug/debugger_panel.html
